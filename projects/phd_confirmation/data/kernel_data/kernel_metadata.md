# Kernel Data Metadata

Generated by `Existing Theory/generate_vwa_kernel_data.m`.

## Exact Kernel Source

The exact second-order sum-interaction eta kernel is evaluated through `computational_efficiency/coeffsMF12.m`, using `coeffsMF12(order=2, g, h, a, b, kx, ky, Ux, Uy)`.

The code implements Madsen & Fuhrman (2012) irregular-wave theory with corrections noted in Fuhrman et al. (2023). For two collinear positive wavenumbers `km` and `kn`, the script calls:

```matlab
a = [1, 1]; b = [0, 0]; kx = [km, kn]; ky = [0, 0];
coeffs = coeffsMF12(2, g, h, a, b, kx, ky, 0, 0);
G_exact = coeffs.G_npm(1) * coeffs.A_npm(1);
```

`coeffs.G_npm(1)` is the `n+m` sum-frequency interaction because `coeffsMF12.m` loops over `pm = [1 -1]`; index 1 is sum and index 2 is difference.

The internal transfer-function formula used by `coeffsMF12.m` is `Lambda2` (Eq. 3.18 / Eq. 3.19 comments in the source):

```text
Lambda2 = h/(2*w_n*w_m*beta) * [
    g*alpha*(w_n*(kappa_m^2 + k_n dot k_m) + w_m*(kappa_n^2 + k_n dot k_m))
  + gamma*(g^2*(k_n dot k_m) + w_n^2*w_m^2 - w_n*w_m*w_sum^2)
]
```

## Sign, Normalization, and Units

The CSV values are signed eta-kernel coefficients per first-order amplitude product. For the unit-amplitude convention used here, `a=[1,1]` and `b=[0,0]`, `coeffs.A_npm(1) = 1/h`, so `G_exact = coeffs.G_npm(1)/h`.

In `surfaceMF12.m`, the corresponding eta contribution is:

```matlab
eta = eta + coeffs.G_npm(cnm) * coeffs.A_npm(cnm) * cos(theta_n + theta_m)
```

Thus, if physical amplitudes are supplied in metres, this kernel multiplies `a_m*a_n` and has units of `1/m`. No absolute value is applied in the CSV files; errors are computed from signed kernel values.

## Interaction Type

The data correspond to surface elevation `eta^(22)` sum-frequency superharmonics only. Difference-frequency terms, velocity-potential kernels, and higher-order kernels are not exported.

## VWA Convention

The exported VWA approximation is:

```text
G_vwa(km, kn) = 0.5 * [G_exact(km, km) + G_exact(kn, kn)]
```

Here `G_exact(k,k)` is evaluated using the same pair-sum convention as the off-diagonal kernel: two equal-but-distinct components with `kx=[k,k]`, not the separate MF12 self-self storage field `G_2*A_2`. With that convention this is equivalent to the local VWA formula used in existing comparison scripts:

```text
f(km) + f(kn), where f(k) = k * (3 - tanh(kh)^2) / (4 * tanh(kh)^3)
```

## Sampling

- Positive collinear wavenumbers only: `ky = [0,0]`, `km > 0`, `kn > 0`.
- CSV columns `km`, `kn`, and `k` are nondimensional ratios: `k/kp`.
- Physical peak wavenumber: `kp = 0.0279 1/m`.
- Gravity: `g = 9.81 m/s^2`.
- Sampled range: `k/kp in [0.2, 2.5]`.
- Grid size: `200 x 200`.
- Slice fixed `kn/kp` values: `0.5` `1.0` `1.5` .
- Invalid or numerically unstable values are written as `NaN`; the selected range avoided the singular point `k=0`.

## Water-Depth Cases

- `k_p h = 0.5`, exported as `kph_0p5`, with `h = 17.9211 m`.
- `k_p h = 1.0`, exported as `kph_1p0`, with `h = 35.8423 m`.
- `k_p h = 5.0`, exported as `kph_5p0`, with `h = 179.211 m`.

## Recommended Slide Cases

- Use `k_p h = 0.5` to show the challenging shallow/intermediate finite-depth case where differences are most visible.
- Use `k_p h = 5.0` as the cleaner deep/intermediate-deep contrast where VWA and exact are visually closer over most of the sampled range.
